O Papel do HTTP

O HTTP (Hypertext Transfer Protocol) é o protocolo fundamental da Web. Ele atua como um mensageiro estabelecendo as regras de como os navegadores (clientes) devem pedir documentos e como os servidores devem devolvê-los. Sem ele, a transferência de páginas HTML, imagens e vídeos na Web não seria padronizada.

Comunicação: Cliente, Servidor, Requisição e Resposta

A Web funciona num modelo Cliente-Servidor. O Cliente (seu navegador) inicia a comunicação enviando uma Requisição (pedido) pela rede. O Servidor (computador remoto onde o site está hospedado) recebe essa requisição, processa os dados e envia de volta uma Resposta, que contém o recurso solicitado ou uma mensagem de erro.

Métodos GET e POST

  • GET: Solicita a representação de um recurso específico (como ler uma página). Os dados enviados vão anexados diretamente na URL. Deve ser usado apenas para buscar dados.
  • POST: Envia dados ao servidor para processamento (como enviar um formulário de login). Os dados vão ocultos no corpo da requisição, proporcionando maior segurança.

Exemplo de Requisição HTTP

GET /contato.html HTTP/1.1
Host: www.site-exemplo.com.br
User-Agent: Mozilla/5.0
Accept-Language: pt-BR

Exemplo de Resposta HTTP

HTTP/1.1 200 OK
Date: Mon, 29 Jun 2026 14:00:00 GMT
Content-Type: text/html
Content-Length: 1045

<!DOCTYPE html>
<html>
...
</html>

Principais Códigos de Status HTTP

Os servidores respondem com códigos padronizados para indicar o resultado da requisição:

  • 200 OK: A requisição foi bem-sucedida e o recurso foi entregue.
  • 301 Moved Permanently (ou 302 Found): O recurso mudou de lugar e o servidor está redirecionando o cliente para a nova URL.
  • 400 Bad Request: O servidor não entendeu a requisição do cliente devido a uma sintaxe inválida.
  • 403 Forbidden: O cliente foi autenticado, mas não tem permissão para acessar o recurso solicitado.
  • 404 Not Found: O servidor não conseguiu encontrar o recurso solicitado na URL especificada.
  • 500 Internal Server Error: Ocorreu um erro inesperado no lado do servidor ao processar o pedido.
